## A. 憲法追記：他者のmoodを編み込む「意図的な例外」

### 前提となる既存原則（v2.2）

- no social features / no gamification
- プロダクトの土台は「誰にも見られない信頼」
- 義務化装置（通知・ストリーク・空白の可視化）の禁止

### なぜこの機能だけが例外として許されるのか

他者の気配を感じられる時点で広義のソーシャル要素である。それでも許容するのは、
以下の**4つの不在**がすべて成立する場合に限る：

1. **関係の不在** — フォロー・返信・リアクション・比較・カウント・特定。
   どの経路も存在せず、双方向性がゼロ。「見た」ことは相手に伝わらない。
2. **内容の不在** — 文字・ユーザー名・場所・細かい時刻は一切出ない。出るのは色だけ。
3. **義務の不在** — 更新を通知しない。新着を強調しない。追いかける対象にならない。
4. **フィードバックの不在** — 自分の色が誰かにどう見えたかは、決して返ってこない。

位置づけは**ソーシャル機能ではなく「環境光」**。
銭湯の湯気の向こうに誰かがいる。誰かは分からないし、話しかけることもできない。
それでも、ひとりで書いているのではないという温度だけが伝わる。

> **一文の憲法：他者は気配として編まれる。関係としては編まれない。**

### 例外が破られる兆候（将来これをやったら原則違反）

人数の表示／新着の強調／特定の色を追える導線／リアクション／
「あなたの色が◯人に見られました」の類。ひとつでも入れたらこの例外は成立しない。

## B. 軽量版の設計：他者のmoodを背景に薄く流す

### API：`GET /api/amimono/ambient`

返すのは **hex色の配列だけ**。`{"colors":["#B5543A","#5C7FA6",...]}`

- **集計**：直近72時間の全ユーザーの `letters.seal_color`（NULL除く）。本文・メタデータは読まない。
- **匿名性の担保**：
  - **k匿名**：窓内の件数が k=8 未満なら空配列を返す（過疎時間帯に「この一色はあの人」が成立するのを防ぐ）
  - **色の量子化**：v3.7以降の色は手染め9スウォッチ由来なので指紋性が低い。それ以前の自由色データはスウォッチ最近傍へ丸めてから返す
  - **順序シャッフル・上限200件**。タイムスタンプ・id・地域は載せない
  - **自分の色も含めて返す**（「自分以外」を返すと、自分の投函直後に差分で他者の一通が特定できてしまうため）
- **更新頻度**：ページロード時に1回のみ。サーバ側は全ユーザー共通の集計を**10分キャッシュ**（DB負荷は最大1クエリ/10分）。

### 表示

- 編み物の**地の糸（緯糸）**として、q-gridの背後に横に走る細い糸（1〜2px）を数十本流す。
  opacity 0.06〜0.10。位置はセッション毎にランダム。ホバー等の対話は一切なし。
- 自分のパッチが主役、他者は布の地。凝視しないと気づかない程度が正しい濃度。
- 添える言葉（案・要相談）：「この布の地には、いま同じ空の下で綴られた誰かの色が、名もないまま編み込まれています。」

## C. ゴミ箱廃止 → 二段階の溶解（確定：時間差方式）

### 確定した骨子（2026-07-22 メモ第2版）

- **即時統合案は不採用**。理由：①「開けば読める」思想の即時反転 ②「ひろげて、書きつづける」
  ③TypeTrace再生——②③は色に還元した時点でデータ構造として不可能になる技術的制約。
- **フェーズ1（破棄直後〜7日間・「揺らいでいる」状態）**：現行機能を維持
  （本文が読める・TypeTrace再生・ひろげて書きつづける）。名称と見た目を「ゴミ箱」から変更。
  期間中に「もう戻らない」を明示選択すれば即フェーズ2へ。
- **フェーズ2（7日経過または明示選択・不可逆の溶解）**：本文・TypeTrace・復元導線を完全破棄。
  moodカラーのみが「言葉の編み物」に匿名の色片として溶け込む。
- 世界観整合：tayoriのコアは「時間差で届く／変わる」（封書・天候配送・感光インク）。
  破棄した言葉も即座には消えず、時間をかけて編み物へ還っていく——既存の世界観と一貫。

### C-1. フェーズ1のUI名称・見た目（要ユーザー確定）

名称案（「保留」はプロダクト用語すぎるため対案を追加）：

| 案 | 名称 | 根拠 |
|---|---|---|
| A | **ほどけるまで** | 7日で糸へほどける物理とそのまま接続。時間差のニュアンスが名前に入る |
| B | **揺らぎの間（ま）** | メモ原案。「間」が部屋と時間の二重の意味を持つ |
| C | **手放した言葉** | 行為の名。溶解後の色片の呼び名（還った色）と連続する |

見た目の方向：籠に紙玉が堆積する現行表現をやめ、
**机の隅に置かれた紙片が、日ごとに縁からほどけていく**表現へ。

- ほどけ具合＝経過日数/7 を紙片の縁のフリンジ（ほどけた糸）で無言に示す。
  **数字のカウントダウンは出さない**（義務化装置の禁止。「あと◯日」で毎日開かせる装置にしない）。
- 残り2日を切った紙片を開いた時だけ、「もうすぐ、編み物に還ります。」を一行静かに添える（要判断 → D-2）。
- 入口は現 binBtn の位置（年表タブ）を維持し、アイコンを籠→ほどけかけの紙片に差し替え。

### C-2. 「もう戻らない」明示選択のUI

- 配置：紙片を開いた画面（現 trashOpenOv 相当）の下部。
  「ひろげて、書きつづける」（戻る側）と対になる位置に「もう、戻らない」（還す側）。
- 確認ダイアログ（必須・一括破棄と同じ原則）文言案：

  > この言葉は、いま編み物に還ります。
  > 文字と筆跡はほどけて、色だけが残ります。
  > もう、読み返すことはできません。
  >
  > ［還す］［まだ、置いておく］

- 実行は溶解バッチと**同一の関数**を即時呼び出し（経路を一本化しテスト面を減らす）。

### C-3. フェーズ1→2 移行バッチ

- **常駐ループ**：既存 `start_notifier()` の maintenance_loop に相乗りし、
  `dissolve_due()` を**1時間に1回**ガード付きで実行（前回実行時刻をメモリ保持。専用スレッドは増やさない）。
- **読み取り時の遅延溶解（保険）**：`GET /api/trash` と `GET /api/archive` のハンドラ冒頭で
  `dissolve_due(user_id)` を呼ぶ。Renderのスリープ等でループが止まっていても、
  **ユーザーが見た瞬間には必ず正しい状態**になる。二重実行は同一トランザクション+WHERE条件で冪等。
- 判定は既存流儀に合わせ naive ローカル時刻のISO文字列比較：
  `created_at <= (now - 7日)` 。
- 処理は `_WRITE_LOCK` 下の1トランザクション：
  ```sql
  INSERT INTO woven_scraps (id, user_id, mood_color, woven_month)
    SELECT <新規発番>, user_id, mood_color, substr(created_at,1,7)
    FROM unemptyable_trash WHERE created_at <= :cutoff;
  DELETE FROM unemptyable_trash WHERE created_at <= :cutoff;
  ```

### C-4. DB設計（採用：別テーブル複製 → 元行を物理削除）

```sql
CREATE TABLE woven_scraps (
  id          TEXT PRIMARY KEY,  -- 新規発番。元のidは持ち越さない（元レコードへの経路を断つ）
  user_id     TEXT NOT NULL,     -- 自分の編み物に出すために必要（他者には一切出ない）
  mood_color  TEXT,              -- これが本体。NULL＝色を選ばず捨てた言葉→無彩の色片として編む
  woven_month TEXT NOT NULL      -- 'YYYY-MM'。月ごとの行に置くための粗い時刻のみ
);
```

- **物理削除範囲**：`unemptyable_trash` の該当行ごと DELETE（本文・TypeTrace・縦横・散乱座標すべて消滅）。
  写真・声は一括破棄時点で既に非保持（メモ第3項・確定）。
- **in-place方式（本文をNULL化して同居）は不採用**：「このテーブルにあるものは読める」という
  不変条件が崩れ、読み側全所に分岐が漏れるため。テーブルが分かれていれば型が嘘をつかない。
- `char_count` は保持しない（メモ「moodカラーの値のみ」に従う）。
  帰結として溶解済み色片のパッチは**最小1マス固定**＝「言葉のかさは、ほどけた時に失われる」。
- **恒久ルールの改定**：旧「/api/trash に DELETE/UPDATE は作らない」→
  新「**ユーザー任意の削除APIは今後も作らない。7日の溶解（バッチ＋『もう戻らない』）だけが唯一の出口**」。

### C-5. 編み物内での視覚区別（要否 → **区別する**）

| 種類 | 表現 |
|---|---|
| 現役の便り（開封済） | 通常パッチ：大きさ=字数・気分の色・布の照り |
| 未開封 | 既存の編み目グレー（変更なし） |
| 溶解済み色片 | **最小1マス**＋彩度をわずかに落とす（saturate(.6)）＋「解れ」パターン（斜めのほどけ糸のSVG pattern・未開封の weavePat とは別物）。ホバーのささやきは**無し**（地名も日付も、そもそも持っていない） |

三者が同じ一枚の布に違う質感で同居する＝「そこに書かれた事実はあったが、内容はもう読めない」。

### C-6. API変更

- `GET /api/archive`：`woven_scraps` を `{kind:"scrap", mood_color, woven_month}` として追加返却。
  土地ごとビューでは「還った色」クラスタとして末尾に。
- `POST /api/trash/<tid>/dissolve`：「もう戻らない」用（確認必須・C-4の改定ルールの範囲内）。
- 一括破棄の一発芝居の器：籠の廃止に伴い、
  「選ばれた封筒がほどけて糸になり、画面下の布へ編み込まれていく」演出へ差し替え。

### C-7. 実装順（推奨）

1. woven_scraps テーブル＋ dissolve_due()（バッチ・遅延溶解・dissolveエンドポイント）
2. 編み物への色片表示（C-5）
3. フェーズ1のUI改装（名称確定後：紙片のほどけ表現・もう戻らないボタン）
4. 一発芝居の器の差し替え・binBtnアイコン差し替え
